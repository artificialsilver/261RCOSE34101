# Assignment 1-2 Report

## Summary
이 과제에서는 Assignment 1-1의 공유 메모리 버퍼를 기반으로 POSIX 세마포어 동기화를 추가했습니다. 세마포어는 단일 슬롯 버퍼의 임계 영역을 보호하여 한 번에 하나의 프로세스만 공유 버퍼 상태를 읽거나 쓸 수 있도록 합니다.

## `s_wait()`와 `s_quit()`를 사용한 위치

- `produce()`
  - `s_wait()`를 호출하여 `is_empty` 상태를 확인하고 쓰기 작업을 시작하기 전에 임계 영역을 잠급니다.
  - `s_quit()`를 호출하여 `message.sender_id`, `message.data`, `account_id`를 모두 기록하고 `is_empty`를 0으로 변경한 뒤 임계 영역을 해제합니다.

- `consume()`
  - `s_wait()`를 호출하여 `is_empty` 상태를 확인하고 메시지를 읽기 전까지 임계 영역을 잠급니다.
  - `s_quit()`를 호출하여 메시지를 읽고 `is_empty`를 1로 업데이트한 뒤 임계 영역을 해제합니다.

## 왜 이 위치가 임계 영역인지

공유 메모리 버퍼는 생산자와 소비자가 동시에 접근하는 공용 자원입니다. 다음 작업은 모두 임계 영역에 속합니다.

- `is_empty` 읽기/쓰기
- `message.sender_id`, `message.data`, `account_id` 쓰기
- 버퍼 상태를 `is_empty`로 변경하는 작업

동기화가 없으면 생산자와 소비자가 같은 시간에 버퍼를 읽고 쓰면서 데이터가 손상되거나 상태가 잘못 해석될 수 있습니다.

## Assignment 1-1에서 발생할 수 있는 레이스 컨디션

Assignment 1-1에서는 생산자와 소비자가 세마포어 없이 버퍼를 직접 접근합니다. 따라서 다음과 같은 문제점이 생깁니다.

- 생산자가 메시지를 쓰는 도중 소비자가 동일한 버퍼를 읽을 수 있음
- 소비자가 `is_empty`를 확인한 뒤 다른 프로세스가 버퍼를 변경하여 잘못된 데이터를 읽을 수 있음
- 생산자가 소비자가 메시지를 읽기 전에 버퍼를 덮어쓸 수 있음

이로 인해 소비자는 불완전하거나 일관성 없는 데이터를 읽을 수 있고, 버퍼 상태는 잘못된 값으로 바뀔 수 있습니다.

## 세마포어 및 정리 동작

- `init_sem()`에서 `sem_open()`을 사용하여 초기값 1로 POSIX 명명 세마포어를 생성/연결합니다.
- `s_wait()`는 `sem_wait()`으로 임계 영역 진입을 직렬화합니다.
- `s_quit()`는 `sem_post()`로 임계 영역을 해제합니다.
- `destroy_sem()`는 `sem_close()`로 세마포어를 닫고 `sem_unlink()`로 제거합니다.
- 공유 메모리 정리는 `munmap()`, `close()`, `shm_unlink()`로 수행합니다.
